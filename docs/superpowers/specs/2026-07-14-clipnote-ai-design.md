# ClipNote AI 产品与技术设计规格

**状态：** 已获用户批准  
**目标平台：** Windows 11  
**产品形态：** 本地优先的桌面剪贴板与便签工作台  
**首版用户：** 产品经理、开发人员等高频处理碎片信息的个人用户  
**视觉方向：** Editorial Desk（编辑部工作桌）

## 1. 产品定义

ClipNote AI 是一个常驻 Windows 桌面边缘的个人信息工作台。它自动保存可授权采集的剪贴板内容，并提供便签、全文搜索、收藏、标签和快捷复用能力；AI 作为用户主动触发的增强层，用于解释命令、总结文字、提取待办和整理便签。

产品首先解决“信息记过却找不到”和“记录过程打断工作”的问题。会议语音、屏幕识别和实时纪要属于后续阶段，不能拖累第一版剪贴板与便签闭环。

### 1.1 产品原则

1. **随手可存：** 新建便签或保留剪贴板内容不超过一次主要操作。
2. **需要即得：** 用户应能在数秒内搜索并重新使用历史内容。
3. **默认本地：** 未主动调用 AI 时，内容不得离开本机。
4. **低干扰常驻：** 入口随时可用，但不持续占用主要工作区。
5. **AI 是增强层：** 离线、无 API Key 时，剪贴板、便签和搜索仍完整工作。
6. **体验即功能：** 视觉、动效、键盘操作和状态反馈属于核心验收项，不是开发结束后的装饰。

## 2. 核心用户痛点

| 痛点 | 产品应对 |
|---|---|
| 命令、代码、链接被新剪贴板内容覆盖 | 自动保存剪贴板历史，并支持收藏和再次复制 |
| 记得过去使用过某条信息，但找不到来源 | 本地全文搜索、类型筛选、标签、来源应用和时间过滤 |
| 不清楚命令用途、参数或风险 | 用户主动触发 AI 解释命令 |
| 在 IDE、浏览器和笔记软件间切换打断思路 | 桌面贴边入口和全局快捷键快速唤起 |
| 临时想法、会议记录和代码片段混在一起 | 统一存储、按内容类型和便签类型区分 |
| 会议过程中难以同时讨论和记录 | 后续会议模式提供转写、截图和结构化整理 |
| 剪贴板可能包含密码、Token 和内部代码 | 来源应用排除、敏感检测、遮罩、AI 发送确认和本地安全存储 |

## 3. 范围与分期

### 3.1 MVP 范围

MVP 必须形成完整的“采集—组织—查找—复用—转便签”闭环：

- 桌面贴边工作台
- 系统托盘与全局快捷键
- 文本、代码、URL、图片和文件路径剪贴板历史
- 去重、来源识别、收藏、置顶和清理
- 普通便签与检查清单
- 标签和本地全文搜索
- 独立桌面便签窗口
- 本地设置、隐私排除、导入与导出
- 用户主动触发的单次 AI 操作

### 3.2 明确不进入 MVP

- 账号注册、云同步和团队协作
- 手机端及 macOS/Linux
- 自动持续录屏和持续 OCR
- 实时录音、语音转写和会议纪要
- 向量数据库和语义搜索
- 浏览器扩展、插件市场和复杂知识图谱

### 3.3 后续会议助手

会议助手作为独立阶段开发，支持：

- 麦克风和 Windows 系统音频采集
- 本地优先、云端可选的实时语音转写
- 用户主动截图或选区识别
- 低频屏幕上下文采样（必须持续显示采集状态）
- 决策、行动项、负责人、截止时间、风险和未决问题提取
- 会议记录与便签互转

不默认上传原始音频或持续屏幕录像。所有采集由用户主动启动，可随时暂停和停止，并提醒用户遵守录音告知要求。

## 4. 功能设计

### 4.1 桌面贴边工作台

- 默认吸附在主显示器右侧，允许切换左侧。
- 收起状态为窄小的“桌面书签/页签”，延续 Editorial Desk 的实体纸张语言。
- 点击、鼠标触碰或全局快捷键后展开。
- 展开时尽量不抢占当前应用焦点；需要输入搜索词时再进入键盘焦点。
- 可拖动并吸附到其他显示器，按显示器与 DPI 保存位置。
- 失焦后自动收起，延迟可配置，也可关闭。
- 系统托盘提供打开、暂停采集、设置和退出。
- 默认快捷键候选：`Alt + Space` 打开工作台，`Alt + Shift + N` 新建便签。注册冲突时提示并提供替代组合，快捷键允许修改。

### 4.2 剪贴板历史

支持以下内容：

- 纯文本和富文本降级后的纯文本
- 代码片段
- URL
- 图片
- 文件路径列表

规则：

- 连续复制相同内容时，不创建重复记录，仅更新最近使用时间和使用次数。
- 根据内容和来源识别文本、代码、链接、图片或路径。
- 展示来源应用、采集时间、类型和内容预览。
- 单击记录写回剪贴板；明确的快捷操作可进一步粘贴到原应用。
- 支持收藏、置顶、删除和批量清理。
- 普通历史默认保留 30 天，收藏永久保留；两者均可配置。
- 图片预览与原图分离存储，并受用户可配置的磁盘空间上限约束。

### 4.3 便签

- 轻量 Markdown 编辑体验，不暴露复杂编辑器控件。
- 支持标题、正文、标签、颜色和置顶。
- 支持普通便签与检查清单。
- 自动保存，无显式保存按钮。
- 任意剪贴板记录可转为便签。
- 可从便签快速复制代码块或选中段落。
- 可创建独立桌面便签，并保存位置和尺寸。
- MVP 不实现历史版本系统；AI 修改不得静默覆盖原文，而应先显示对比，再由用户确认替换或创建新便签。

### 4.4 搜索与组织

- 一个统一搜索框检索剪贴板与便签。
- 使用 SQLite FTS5，基础搜索不依赖 AI。
- 支持关键词、类型、标签、来源应用、收藏状态和时间范围筛选。
- 默认视图为最近使用、收藏、便签和全部历史。
- 完整键盘操作：上下选择，Enter 复制，快捷键打开预览或动作菜单。

### 4.5 MVP AI 动作

AI 不以通用聊天框作为主要形态，而通过当前内容的上下文动作出现：

- 总结长文本
- 解释命令、参数和潜在风险
- 优化便签表达
- 转换为检查清单
- 提取关键事项或行动项
- 推荐便签标题和标签

调用规则：

- 只能由用户主动触发，不后台自动调用。
- 调用前明确展示或说明将发送的内容范围。
- 检测到疑似密码、Token 等敏感内容时默认阻止发送，用户二次确认后才可继续。
- AI 失败不得影响本地内容和基础功能。

### 4.6 设置与隐私

- 开机启动
- 主题、透明度与减少动画
- 贴边方向和自动收起延迟
- 历史保留天数、最大条数和图片空间上限
- 排除应用列表
- 暂停剪贴板采集
- 敏感内容检测开关和行为
- AI 服务开关、模型字段和凭证管理
- 本地数据导入、导出和清理

## 5. 视觉与交互设计

### 5.1 选定方向：Editorial Desk

视觉采用“编辑部工作桌”而非传统后台表单：

- 温暖、低饱和的纸张底色与墨色文字
- 精致的编辑排版和清楚的信息层级
- 适度的纸页、卡片叠放和实体便签触感
- 小面积陶土橙、芥末黄或灰绿作为状态色
- 稳定、克制的阴影和边界，不依赖大面积玻璃拟态
- 标题字体与正文/代码字体形成明确但协调的对比

视觉不能演变为卡通手账。产品仍应传达专业、精准和高效，适合产品经理与开发人员长时间使用。

### 5.2 品牌与动效语言

- 收起状态表现为贴在屏幕边缘的书签页签。
- 刚捕获内容时，页签产生类似纸张轻弹或印章落下的微反馈。
- 展开时由页签自然延展为工作台，建立空间连续性。
- 新记录从边缘入口进入列表；去重更新时反馈发生在已有卡片上。
- 便签从工作台脱离为桌面便签时使用共享元素过渡。
- AI 处理中使用校对、批注或逐行整理的语义，而非通用旋转加载器。
- 动画只用于解释状态、建立空间关系或确认操作；无功能意义的炫技动画必须删除。

### 5.3 关键界面

1. **边缘页签：** 待机、刚采集、暂停、AI 处理中和后续会议采集中等状态。
2. **展开工作台：** 搜索、默认视图、内容列表、快速预览与上下文动作。
3. **聚焦预览：** 阅读完整内容、复制局部、收藏、转便签和调用 AI。
4. **便签编辑：** 以内容为中心，弱化工具条和表单边框。
5. **桌面便签：** 保持编辑部纸张语言，但支持清楚的焦点、拖动和缩放反馈。
6. **设置：** 设置页也需维持品牌排版，不退化为默认表单控件堆叠。

### 5.4 视觉验收

- 核心状态均有设计，包括空数据、加载、错误、离线、权限拒绝和快捷键冲突。
- 展开、收起、捕获、搜索、复制、保存和 AI 处理使用一致的动效语言。
- 100%、125%、150% 和 200% DPI 下保持清晰与稳定。
- 深浅主题分别调校，不采用机械反色。
- 支持减少动画设置、键盘导航、清楚的焦点状态和辅助技术语义。
- 长时间使用不得因高亮、透明或频繁运动造成明显疲劳。
- 每个主要实现里程碑需在真实桌面应用中运行并截图验收，不能只查看组件代码。

## 6. 技术选型

### 6.1 最终技术栈

- **桌面框架：** Tauri 2
- **原生层：** Rust（Windows MSVC 目标）
- **前端：** React + TypeScript + Vite
- **样式系统：** Tailwind CSS 4 或基于 CSS 变量的设计令牌层
- **动效：** Motion/Framer Motion；仅对特殊过渡选择性使用 GSAP
- **可访问组件基础：** Radix Primitives，但必须自定义视觉，不直接套用默认组件外观
- **状态管理：** Zustand（UI 状态）+ TanStack Query（异步命令/数据状态）
- **数据库：** SQLite + FTS5
- **AI：** 通过 Rust 后端的 `IAiAssistant` 等价抽象接入，首个适配器可对接 Claude API

### 6.2 选型理由

Tauri 提供 Windows 原生外壳、系统托盘、全局快捷键和低资源占用；React/CSS 动画体系能更高效地实现高完成度的视觉、排版与微交互。Rust 层负责系统权限、剪贴板、窗口、数据和密钥边界，避免敏感凭证进入 WebView 前端。

WPF 不再作为首选，因为当前项目明确将高水准交互与视觉迭代速度置于核心位置。Electron 仍不选择，原因是长期驻留工具的内存和包体成本更高，而当前机器已具备 Tauri 所需的 Rust、Node、Visual Studio C++ 工具链和 WebView2。

### 6.3 本机环境结论

已实测本机具备：

- Node.js 24.15.0、npm 11.12.1、pnpm、Bun
- Rust/Cargo 1.97.0，目标为 `x86_64-pc-windows-msvc`
- Visual Studio 2022 Community 和 MSVC 14.44 工具链
- Windows SDK
- Microsoft WebView2 Runtime 150.0.4078.65
- Git 2.54.0

本机未安装 .NET SDK，但最终方案不依赖 .NET。无需重装整套开发环境；实施前需运行一个最小 Tauri 编译探针，并仅在构建日志明确指出缺失组件时精准补装。项目级 npm 和 Cargo 依赖仍需正常安装。

## 7. 架构

采用模块化单体：

```text
ClipNote
├── apps/desktop                 # React/TypeScript UI
│   ├── shell                    # 边缘页签、展开工作台、托盘状态映射
│   ├── clipboard               # 历史列表、预览和动作
│   ├── notes                   # 便签与桌面便签 UI
│   ├── search                  # 搜索与筛选
│   ├── ai-actions              # AI 对比、确认和结果呈现
│   └── settings                # 设置与隐私
├── src-tauri                    # Rust/Tauri 原生层
│   ├── clipboard               # Windows 剪贴板监听与规范化
│   ├── windows                 # 贴边、焦点、多显示器和快捷键
│   ├── storage                 # SQLite、FTS5、文件资源和清理
│   ├── security                # 敏感检测、来源排除和凭证
│   ├── ai                      # AI 提供商抽象与网络调用
│   └── commands                # 向前端暴露的最小类型化命令
└── tests
    ├── unit
    ├── integration
    └── e2e
```

边界要求：

- React 层不直接访问数据库、文件系统、Windows API 或 AI 密钥。
- Rust 命令接口必须类型化、最小化，并验证来自 WebView 的所有输入。
- 领域规则（去重、保留策略、敏感判定）不与具体 UI 组件耦合。
- AI 提供商可替换，基础功能不得依赖其可用性。

## 9. 核心数据流

### 9.1 剪贴板采集

```text
Windows 剪贴板变化
→ 获取来源应用和内容
→ 检查排除规则
→ 规范化内容
→ 敏感信息检测
→ 内容哈希与去重
→ 写入 SQLite/资源文件
→ 通过事件通知 UI
→ 边缘页签与列表执行状态反馈
```

### 9.2 搜索与复用

```text
用户输入关键词或筛选条件
→ Rust 查询 SQLite FTS5
→ 返回分页、类型化结果
→ React 呈现并支持键盘选择
→ 用户复制或粘贴
→ 更新最近使用时间和次数
```

### 9.3 AI 动作

```text
用户选择内容和 AI 动作
→ 本地检查敏感性与发送范围
→ 必要时要求二次确认
→ Rust 后端调用配置的 AI 提供商
→ 解析结构化结果
→ 前端显示原文/建议对比
→ 用户确认替换或新建便签
```

## 10. AI 集成策略

MVP 的 AI 功能是总结、解释和结构化提取，使用单次 Messages API 调用即可，不构建自主 Agent。

首个 Claude 适配器应：

- 使用官方 Anthropic SDK（若 Rust 无官方 SDK，则通过受控的后端 HTTP 适配层实现，并保持提供商接口独立）。
- 默认模型配置为 `claude-opus-4-8`，同时允许用户在设置中修改受支持的模型 ID。
- 对标题、标签、检查清单和行动项使用结构化输出。
- 长输出使用流式传输，但 UI 主要呈现最终可确认结果。
- 使用类型化错误处理，区分认证、限流、网络和服务端错误。
- 绝不在日志、前端状态或崩溃报告中写入 API Key 或完整敏感正文。

会议阶段的语音识别采用独立的 `ITranscriptionProvider` 等价接口，优先本地转写，云端由用户主动选择。会议总结发送转写文本和用户选择的截图，不默认发送原始音频。

## 11. 异常与安全

- 剪贴板暂时被占用：短暂退避重试，不阻塞 UI。
- 图片过大：生成缩略图，并受空间上限和清理策略管理。
- 数据库异常：保留备份，进入可解释的恢复或只读状态。
- 快捷键冲突：明确提示并提供替代组合。
- AI 无网络、认证失败或限流：保留用户输入并允许重试，不影响本地数据。
- 敏感内容：默认遮罩预览，并阻止直接发送 AI，除非用户二次确认。
- 来源应用排除：通过前台窗口/进程识别，在持久化前完成阻断。
- 日志：不得包含剪贴板正文、API Key、完整会议文本或未经脱敏的文件路径。
- Tauri 命令：遵循最小权限，限制文件路径到应用数据目录，并验证所有前端参数。

- 多显示器与 100%/125%/150%/200% DPI
- 贴边展开、失焦收起和焦点行为
- 快捷键冲突、托盘和开机启动
- 系统休眠/唤醒与应用崩溃恢复
- 真实应用中的键盘工作流
- 离线和无 AI Key 场景

### 12.3 MVP 指标

- 冷启动到可用目标小于 2 秒。
- 空闲内存目标小于 150 MB。
- 复制后 300 毫秒内出现在历史列表。
- 1 万条文本记录的搜索响应目标小于 150 毫秒。
- 新建便签到可输入不超过一次主要操作。
- 无网络、无 AI Key 时基础功能完整工作。
- 暂停采集后不得新增记录。
- 排除应用中的内容不得写入数据库。
- 核心动效在目标设备上保持流畅，并尊重减少动画设置。

## 14. 已确定决策

- 平台：Windows 个人工具优先。
- MVP：剪贴板历史与便签。
- 常驻方式：屏幕贴边页签，触碰、点击或快捷键展开。
- 数据策略：本地优先，AI 仅主动触发。
- 桌面技术：Tauri 2 + Rust。
- 前端技术：React + TypeScript。
- 视觉方向：Editorial Desk。
- 质量目标：产品必须是一件视觉与交互均有辨识度的作品，而非仅完成表单和功能的软件。
